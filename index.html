O Cenário Atual

A operação hoje
vive em planilhas

A Vitrine Global gerencia toda a operação de importação, vendas e financeiro em planilhas Excel. Isso gera riscos, retrabalho e limita o crescimento.

📊

17 abas de planilha

Cargas, itens, orçamentos, pedidos, financeiro, estoque — tudo separado e sem integração automática.

🔢

Cálculos manuais

Custo unitário BRL, rateio de frete, cotação USD por etapa — tudo calculado manualmente com risco de erro.

🏢

Sem visão multi-filial

Cada filial mantém seus próprios controles. Não há consolidação automática de resultados.

🛒

Marketplace manual

Pedidos do Mercado Livre, Shopee e Amazon são controlados manualmente, sem sincronização de estoque.

08 / 14
Decisões Técnicas

Stack tecnológica escolhida

⚛️
Frontend

React 18 + TypeScript + Vite

Interface rápida, tipada e com hot-reload instantâneo durante o desenvolvimento.

🎨
UI

shadcn/ui + Tailwind CSS

Design system consistente e customizável com a identidade Vitrine Global.

⚡
Backend

Node.js + Fastify

Servidor de alta performance. Fastify é 2× mais rápido que Express.

🔗
API

tRPC

Type-safety total entre front e back. Elimina contratos de API manuais.

🐘
Banco de Dados

PostgreSQL 16 + Drizzle ORM

Arrays nativos, JSONB, colunas geradas e performance analítica para o DRE.

🔐
Auth

JWT + bcrypt Local

Sem OAuth externo. Controle total sobre sessões e fluxo de acesso.

📬
Filas

BullMQ + Redis

Sync de marketplaces assíncrono sem travar a interface do usuário.

📦
Monorepo

Turborepo

Tipos TypeScript compartilhados entre front e back. Mudança no schema reflete automaticamente.

🚀
Deploy

Railway → AWS ECS

Deploy simples no início. Migração para AWS ECS quando necessário escalar.

Arquitetura

Multi-Filial
nativo

Toda tabela operacional possui empresa_id e filial_id. O contexto é injetado automaticamente via JWT, garantindo isolamento total dos dados.

Login

→
Middleware
Injeta contexto
→
PostgreSQL
WHERE filial_id = ?
